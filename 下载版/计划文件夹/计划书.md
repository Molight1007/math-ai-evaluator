## 一、比赛核心要求提炼

### 1.1 初赛要求（9月5日前提交）

构建一个**基于 Intern-S1 API 的单智能体系统**，具备：

| 能力 | 说明 |
|------|------|
| ✅ 推理求解 | 理解自然语言数学问题，自主规划思路并求解 |
| ✅ 过程解释 | 以启发式表达方式解释推理过程 |
| ✅ 结构化输出 | **必须输出 JSON 格式结果** |
| ✅ 多类型稳健 | 覆盖偏微分方程、复分析、拓扑学、运筹学等 **18 个子领域**，共 **112 道题** |

### 1.2 总决赛要求

在初赛基础上扩展为 **多智能体协作系统**：
- 可交互 Demo（Web/Notebook）
- 超长程推理支持
- 自动题型识别 → 调用不同 solver agent
- 引入过程校验智能体，实现推理自主调控

### 1.3 评分标准（总分100分）

| 维度 | 权重 | 评分方式 | 关键点 |
|------|------|----------|--------|
| **答案正确性** | 60% | 客观线性计分 | 正确率 × 60，**必须输出 JSON** |
| **推理策略与系统设计** | 10% | 主观分档 | 模块设计、推理链组织、协作逻辑 |
| **展示质量** | 20% | 主观分档 | Demo、视频、报告完整性 |
| **创新性与可扩展性** | 10% | 主观分档 | 机制创新、可迁移性、推广潜力 |

### 1.4 关键约束

- ❌ **不允许**：仅通过提示词堆叠或固定模板完成单轮作答
- ❌ **不允许**：人工逐题干预作答、赛后补填结果、伪造日志
- ✅ **鼓励**：问题理解 → 推理规划 → 过程校验 → 解释生成 → 多智能体协作
- ✅ **必须**：输出结构化 JSON + 日志信息

## 三、核心挑战分析

### 3.1 比赛的本质是什么？

这是一个**智能体（Agent）设计比赛**，不是模型微调比赛。核心在于：

> **如何设计一个能自动解题、展示推理过程、并输出结构化结果的 Agent 系统。**

Intern-S1 本身已在 CMO（全国中学生数学奥林匹克）中以 102 分位列大模型榜首，说明**模型数学能力本身很强**。比赛的难点在于：

1. **题目覆盖极广**（18 个子领域 × 112 道题），单一 Prompt 很难通用
2. **必须展示推理过程**，不能只给答案
3. **必须结构化输出 JSON**，格式错误直接判未通过
4. **不能只靠 Prompt 堆叠**，需要真正的 Agent 设计

### 3.2 我们的核心策略

```
┌──────────────────────────────────────────────────────────────┐
│                    数学智能体系统架构                          │
├──────────────────────────────────────────────────────────────┤
│                                                              │
│  用户输入数学问题（自然语言）                                  │
│       │                                                      │
│       ▼                                                      │
│  ┌─────────────┐                                             │
│  │ ① 题型分类器 │ ← 自动识别：PDE / 复分析 / 拓扑 / 代数...    │
│  │   (Agent 1)  │                                             │
│  └──────┬──────┘                                             │
│         │                                                     │
│         ▼                                                     │
│  ┌─────────────┐                                             │
│  │ ② 知识检索   │ ← IMA 知识库召回：相关方法 + 公式 + 例题     │
│  │   (RAG 层)   │                                             │
│  └──────┬──────┘                                             │
│         │                                                     │
│         ▼                                                     │
│  ┌─────────────┐                                             │
│  │ ③ 推理规划   │ ← CoT + 演绎推理 + 知识增强 Prompt          │
│  │   (Agent 2)  │                                             │
│  └──────┬──────┘                                             │
│         │                                                     │
│         ▼                                                     │
│  ┌─────────────┐                                             │
│  │ ④ Intern-S1  │ ← API 调用，不改变模型                      │
│  │   API 推理   │                                             │
│  └──────┬──────┘                                             │
│         │                                                     │
│         ▼                                                     │
│  ┌─────────────┐                                             │
│  │ ⑤ 过程校验   │ ← 自验证：代入检查、量纲验证、逻辑审查       │
│  │   (Agent 3)  │                                             │
│  └──────┬──────┘                                             │
│         │                                                     │
│         ▼                                                     │
│  ┌─────────────┐                                             │
│  │ ⑥ JSON 格式化│ ← 严格按比赛要求输出结构化结果               │
│  └──────┬──────┘                                             │
│         │                                                     │
│         ▼                                                     │
│  输出：JSON { answer, reasoning, steps, verification }       │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

**核心思路：**
- 不改变 Intern-S1 模型参数
- 通过 **Agent 设计**（多模块协作）+ **RAG 知识库**（领域知识注入）+ **Prompt Engineering**（推理链引导）提升解题效果
- 这恰好符合比赛「不鼓励仅通过提示词堆叠，鼓励智能体设计创新」的精神
