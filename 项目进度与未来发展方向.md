## 一、项目概览

本项目构建了一套完整的**数学智能体评测与推理系统**，以 Intern-S1（书生浦语）作为推理引擎，DeepSeek 作为评判引擎，Lean 4 作为形式化验证工具，实现了从**多格式题目输入 → AI 推理 → 自动评判 → 形式化验证 → 可视化报告**的全链路闭环。

## 三、各模块详细进度

### 3.1 评测基础设施（完成度 95%）✅

**已完成功能：**

- **格式转化工具**（`转化工具/`）：支持 PDF / Word / PPT / Markdown / Excel → JSON 自动转化，含章节标题识别、选择题合并、答案区域跳过
- **命令行入口**（`main.py`，949 行）：三种模式（文件评测 / 答案导入 / 统计查询），7 种格式自动识别
- **GUI 图形界面**（`launcher.py`，约 64KB）：三大标签页（快速评测 / 题库管理 / 题库浏览），支持文件拖放
- **Intern-S1 推理**（`intern_s1.py`）：分步推理 + 自验证，输出结构化 JSON
- **DeepSeek 评判**（`deepseek.py`，401 行）：支持单题评判和批量评判两种模式，批量模式减少约 45% API 调用
- **LLM 客户端**（`llm_client.py`）：OpenAI 兼容的异步 HTTP 客户端，指数退避重试，三级 JSON 提取
- **结果聚合**（`aggregator.py`）：合并推理 + 评判为最终结果，计算统计摘要
- **报告生成**（`reporter.py`，约 48KB）：输出 JSON / CSV / 终端 / HTML 四种格式，HTML 含可视化看板（准确率、领域分布、逐题详情、耗时、Token 消耗）
- **Windows 一键启动**（`启动评测器.bat`）

**待优化：**
- 报告生成器代码较长（48KB），可考虑拆分模块
- 并发控制可以更精细化（按 API 限流动态调整）

### 3.2 题库管理系统（完成度 90%）✅

**已完成功能：**

- **SQLite 题库 CRUD**（`question_bank.py`，约 38KB）：创建/删除题库、手动添加题目、批量导入（JSON）
- **AI 质量审核**：DeepSeek 自动审核题目质量，删除无效题并补全高质量新题
- **随机选题评测**：按数量随机抽取题目，支持领域筛选
- **答案提取**（`answer_extractor.py`，约 19KB）：从 PPT / Word / TXT / PDF / CSV / Excel / JSON 中提取题目-答案对
- **答案匹配**（`answer_matcher.py`，约 8KB）：使用 DeepSeek 语义匹配将答案文档条目关联到题库题目
- **题库浏览**：搜索、筛选、编辑、删除题目

**待优化：**
- 题库代码较长（38KB），可考虑拆分领域管理模块
- 尚不支持批量导出带答案的题库

### 3.3 Lean 4 形式化验证（完成度 80%）⚠️

**已完成功能：**

- **三阶段异步流水线**（`lean_verifier.py`，约 44KB）：
  - 阶段一：DeepSeek 将自然语言推理转化为 Lean 4 代码
  - 阶段二：subprocess 调用 `lake env lean` 编译验证
  - 阶段三：DeepSeek 分析编译错误，区分"转化错误"与"逻辑错误"
- **AND-OR DAG 蓝图分解**（`models.py`）：将证明分解为子引理的树形结构
- **Lean 不可用时自动降级**为纯 LLM 逻辑审查模式
- **test_mathlib 环境**：完整的 Lean 4 + Mathlib4 项目（847 个文件），已成功编译出可执行文件
- **错误根因分析**：定位逻辑断点，生成修正提示词（可反馈给 Intern-S1 重新推理）
- **sorry 检测**：统计不完整证明，标记 incomplete 状态

**这是本项目的特色创新点**，目前国内类似比赛中极少有团队实现形式化验证闭环。

**待完善：**
- 转化成功率需要实测统计（自然语言 → Lean 代码的转化难度较高）
- 编译超时处理可优化
- 批量验证性能可提升
