## 一、项目概述

### 1.1 项目名称

**MathPilot — 基于 Intern-S1 的多智能体数学推理与验证系统**

### 1.2 一句话简介

一个基于 Intern-S1 大模型的多智能体协作系统，能够自动理解、推理、校验数学问题，并以结构化 JSON 输出完整推理过程，覆盖偏微分方程、复分析、拓扑学等 18 个高等数学子领域。

### 1.3 项目定位

本项目旨在构建一个**全链路数学智能体系统**，实现从「问题理解 → 题型分类 → 知识检索 → 推理求解 → 多重校验 → 结构化输出」的完整闭环。不同于简单的 Prompt 堆叠方案，我们设计了**多 Agent 协作架构**，并创新性地引入 **Lean 4 形式化验证**作为过程校验手段，确保推理结果的可靠性和可解释性。

## 二、项目背景

### 2.1 赛事背景

2026 年挑战杯「揭榜挂帅」专项赛由中国共青团中央、中国科协、教育部、全国学联共同主办，聚焦新一代信息技术、人工智能等 8 大前沿领域。上海人工智能实验室基于其科学多模态大模型 Intern-S1 在 2025 年 CMO 中斩获 102 分（大模型榜首，远超金牌线）的卓越表现，发起了本赛题。

### 2.2 核心挑战

- 覆盖 **18 个高等数学子领域**，**112 道私有闭源试题**
- 要求 Agent 自主规划推理路径，**不能仅靠 Prompt 堆叠**
- 必须输出**结构化 JSON**，格式错误直接判未通过
- 评测过程**不可见**：无法获取测试题、标准答案、逐题反馈
- 提交次数有限：每日 2 次，每周最多 10 次

### 3.2 核心模块

#### 模块一：题型分类 Agent

- **功能**：自动识别 18 个数学子领域（PDE、复分析、拓扑学、运筹学等）
- **技术**：基于 DeepSeek 的语义分类，支持多标签和置信度评估
- **输出**：领域标签 + 推荐推理策略 + 知识库检索关键词

#### 模块二：RAG 知识检索层

- **功能**：根据题型分类结果，从 IMA 知识库召回相关解题方法、公式、例题
- **技术**：基于 Intern-S1 API 的语义检索，结合结构化知识库索引
- **知识库规模**：覆盖 18 个子领域，每领域含方法论文档、典型例题、公式手册

#### 模块三：推理求解 Agent

- **功能**：基于领域知识 + 长 CoT 提示进行分步推理求解
- **技术亮点**：
  - 参考「Towards Reasoning Era」论文设计长推理链模板
  - 参考「Deductive Beam Search」实现演绎推理搜索
  - 领域自适应 Prompt + Few-shot 示例注入
  - 支持 AND-OR DAG 蓝图分解（将证明拆解为子引理树）

#### 模块四：多重过程校验系统 ★核心创新

| 校验层 | 方式 | 说明 |
|--------|------|------|
| **第一层：自验证** | 代回原题检算 | 将推理结果代入原始问题验证 |
| **第二层：DeepSeek 评判** | LLM 逻辑审查 | 独立模型检查推理链的逻辑一致性 |
| **第三层：Lean 4 形式化验证** | 定理证明器验证 | **国内同类比赛首创**，将推理过程转化为 Lean 4 代码，通过编译器严格验证 |

#### 模块五：JSON 格式化输出

- 多层级 JSON 提取（正则 → JSON 解析 → 字段校补）
- 自动修复常见格式错误
- 确保 `final_response` 非空且符合平台规范

### 3.3 Agent 编排器（Orchestrator）

协调多 Agent 协作的核心调度模块：
- 管理 Agent 间的数据流转
- 支持校验失败时的自动重试和回溯
- 全局异常捕获，单题失败不影响整体
- 完整的 trace 日志记录（用于同分场景提升设计评分）

## 五、创新亮点

### 5.1 Lean 4 形式化验证闭环 ★独创

本项目是国内同类参赛作品中**首个**引入形式化验证的团队。通过三阶段流水线将数学推理转化为可编译验证的 Lean 4 代码：

```
自然语言推理 → DeepSeek 转化 Lean 代码 → lake env lean 编译验证 → 错误根因分析
```

这一创新使推理结果的可靠性有了**数学级别的保证**，而非仅依赖模型的「自信」。

### 5.2 多 Agent 协作架构

不同于单模型 Prompt 堆叠方案，我们设计了完整的 Agent 协作体系：
- **题型分类 Agent** — 智能路由到正确的知识库和 Prompt
- **RAG 知识检索** — 领域知识注入，弥补模型知识盲区
- **推理求解 Agent** — 长 CoT + 演绎搜索
- **过程校验 Agent** — 三重验证确保正确性

### 5.3 AND-OR DAG 证明树

将复杂证明分解为子引理的有向无环图结构，支持：
- 并行验证子引理
- 证明失败的精确回溯
- 推理过程的可视化展示

### 5.4 完备的工具链生态

项目不仅是一个解题 Agent，更构建了一整套**开发-评测-可视化**工具链：
- 7 种格式自动转化（PDF/Word/PPT/Markdown/Excel/JSON/CSV）
- GUI 图形界面（一键评测、题库管理、报告浏览）
- HTML 可视化报告（准确率、领域分布、逐题详情）
- Docker 一键部署（确保环境零问题）

## 七、比赛提交策略

### 7.1 提交规划

充分利用每周 10 次（每日 2 次）的提交额度，分阶段提交：

| 阶段 | 批次 | 策略 |
|------|:--:|------|
| **试探期**（9/1-9/5） | 前 4-6 次 | 验证平台兼容性，排查格式/依赖/超时问题 |
| **调优期**（9/6-9/10） | 中 6-8 次 | 根据初步反馈调整 Prompt、优化性能 |
| **冲刺期**（9/11-9/15） | 后 4-6 次 | 提交最优版本，争取得分最大化 |

### 7.2 风险防控

- 每次提交前本地完整模拟全流程（依赖安装→加载→推理→输出）
- Docker 化确保线上环境与本地一致
- 全局异常捕获 + 降级策略（Lean 不可用时自动切换纯 LLM 校验）
- 保留历史版本，可快速回滚
